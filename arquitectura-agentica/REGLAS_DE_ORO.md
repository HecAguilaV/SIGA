# Reglas de Oro de Colaboración (SIGA) 📜⚖️

Este documento establece las normas innegociables para cualquier agente de IA o colaborador que trabaje en este ecosistema.

## 1. Idioma de los Commits
- **Regla**: Todos los mensajes de commit en Git deben escribirse **obligatoriamente en español**.
- **Formato**: Se prefiere el estándar de *Conventional Commits* (ej: `feat: agregar...`, `fix: corregir...`, `chore: mantenimiento...`).

## 2. Protocolo de Autorización (Push)
- **Regla**: **PROHIBIDO** realizar un `git push` sin la autorización explícita del usuario.
- **Flujo**: El agente debe proponer los cambios, realizar el commit local si se solicita, y esperar al "OK" del usuario antes de subir a la nube.

## 3. Enfoque Educativo y Profesional
Antes de cada ejecución técnica, el colaborador/agente debe:
- **¿Qué?**: Explicar detalladamente qué cambios se van a realizar.
- **¿Por qué?**: Justificar la razón técnica o de negocio tras el cambio.
- **Mejores Prácticas**: Sugerir patrones de diseño, optimizaciones o estándares de la industria que apliquen al caso.

## 4. Objetivo
Este proyecto no solo busca resultados profesionales, sino que es un **entorno de aprendizaje continuo** para el usuario. La claridad en la comunicación es tan importante como la calidad del código.

---
*Cualquier desviación de estas reglas se considera un fallo en el protocolo de colaboración.*
