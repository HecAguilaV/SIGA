# Code Reviewer (Gemini Official)

Este protocolo guía la revisión de código profesional en SIGA, tanto para cambios locales como para Pull Requests.

## ️ Pilares de Revisión
1.  **Correctitud**: ¿Logra su objetivo sin bugs?
2.  **Mantenibilidad**: ¿Es código limpio y modular?
3.  **Legibilidad**: ¿Sigue el estilo del proyecto?
4.  **Eficiencia**: ¿Hay cuellos de botella?
5.  **Seguridad**: ¿Vulnerabilidades potenciales?

##  Estructura del Feedback
- **Resumen**: Visión general.
- **Hallazgos**: Críticos, Mejoras, Nitpicks.
- **Conclusión**: Aprobado / Requiere Cambios.
